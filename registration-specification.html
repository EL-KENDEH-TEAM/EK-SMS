1. Overview

EK-SMS uses a hybrid onboarding approach for school registration. Schools apply through a self-service form, but require platform admin approval before activation.

Why Hybrid?

  • Trust & Verification: As an anti-corruption platform, we need to verify we're onboarding legitimate schools
  • Scalability: Schools initiate the process themselves, reducing manual work
  • Control: Platform admin reviews ensure quality without becoming a bottleneck
  • Accountability: Double verification (applicant + principal) prevents fraudulent applications

High-Level Flow

  1. School representative fills out multi-step application form
  2. Applicant verifies their email
  3. Principal confirms the application (if different from applicant)
  4. Application enters platform admin review queue
  5. Admin reviews, approves or rejects
  6. On approval: School tenant + admin account created automatically

2. Actors

2.1 School Applicant

A person from the school who initiates the registration. Could be:

  • The principal themselves
  • An IT staff member
  • An administrative officer
  • A trusted teacher assigned by the principal

2.2 Principal / Head Teacher

The school authority who must confirm the application. If the principal is also the applicant, this is the same person.

2.3 Platform Admin

EK-SMS team member who reviews and approves/rejects applications. Has access to the admin review dashboard.

2.4 School Admin

The first administrative account created for an approved school. This is either the applicant or the principal, as designated during application.

4. Verification Flow

4.1 Two-Step Verification

All applications go through a two-step verification process to ensure legitimacy:

Step A: Applicant Email Verification

  1. Applicant submits the form
  2. System shows: "Check your email to verify your application"
  3. Applicant receives verification email with unique link
  4. Applicant clicks link â†’ email verified

Step B: Principal Confirmation

  1. After applicant verifies, system sends email to principal
  2. Principal receives confirmation request with application summary
  3. Principal clicks "Confirm" link â†’ application submitted to review queue
Decision: If applicant IS the principal (checkbox checked), only one verification email is sent. The principal email serves as both verification and confirmation.

4.2 Verification Link Expiry

  • Validity: 72 hours from email sent
  • Reminder: Sent at 48 hours if not yet verified
  • After expiry: Applicant must restart the process

4.3 Reminder Schedule

Time Action Recipient
0 hours Initial verification email sent Applicant
48 hours Reminder email sent Applicant (if not verified)
72 hours Link expires, notification sent Applicant

Same schedule applies for principal confirmation after applicant verifies.

6. Admin Review Process

6.1 Review Dashboard

Platform admins access a dashboard showing all applications with:

  • Filters: By status, country, date submitted
  • Search: By school name or email
  • Sort: By date (oldest first recommended so nothing gets stuck)
  • Quick stats: Count per status

6.2 Individual Application View

When reviewing a single application, admin sees:

  • All submitted information (organized by section)
  • Verification status (applicant âœ“, principal âœ“)
  • Submission date and timeline
  • Internal notes field (only visible to admins)

6.3 Admin Actions

Action Description Result
Request More Info Ask applicant for additional details/documents Email sent, status â†’ "More Info Requested"
Approve Accept the application School + admin account created, welcome email sent
Reject Deny the application (reason required) Rejection email sent with reason

6.4 On Approval

When an application is approved, the system automatically:

  1. Creates the school tenant (isolated data space)
  2. Creates the school admin account (applicant or principal as designated)
  3. Generates a temporary password
  4. Sends welcome email with login credentials
  5. Marks application as "Approved"
Security: The temporary password must be changed on first login. If the admin role requires 2FA, user will be prompted to set it up immediately.

6.5 On Rejection

  • Admin must provide a reason (required)
  • Common reasons available as quick-select options plus free text
  • Rejection email sent with the reason
  • School can reapply after 30 days

7. Edge Cases & Decisions

Verification link expires

Decision: Links valid for 72 hours. Reminder sent at 48 hours. After expiry, applicant must restart the entire process.

Principal and applicant have same email

Decision: This means the principal is applying themselves. When "I am the Principal" is checked, only one email is used for both verification and confirmation. Single verification step.

School with same name already exists

Decision: Allow same school name if in a different city. Block if same name AND same city. Show warning: "A school with this name already exists in [city]. Is this a duplicate application?"

Applicant tries to submit another application while one is pending

Decision: One pending application per applicant email per school. Show message: "You already have a pending application for [School Name]. Check your email for the verification link or contact support."

Principal doesn't confirm after several days

Decision: Reminder to principal at 48 hours. After 72 hours, link expires and applicant is notified: "Principal didn't confirm. You may reapply." Applicant may also follow up with principal directly.

Rejected school wants to reapply

Decision: Can reapply after 30 days. They should see the rejection reason so they can address the issues.
